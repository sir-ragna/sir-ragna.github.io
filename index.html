Uitgebreide beschrijving van mijn kennis over ...

Deze lijst is onvolledig en is gedoemd om niet volledig up to date te zijn.

TCP/IP

Ik heb een redelijke kennis van de TCP/IP protocol suite.

HTML & CSS

Een vrij goede kennis van HTML en CSS. Zie ook onder programmeer talen, Javascript.

XML & JSON

Data communicatie door middel van XML en JSON is mij bekend. Mijn kennis van DTD(Document Type Definition) is wel aan het vervagen maar navigeren doorheen zowel XML als JSON zou geen probleem mogen zijn. Ik ben ook op de hoogte van gerelateerde technologieën zoals XQUERY, XPATH, XSLT maar heb hier geen enkele expertise in.

SQL & Relationele Databases

Kennis van niet-platform specifieke SQL. Kan ook simpele ER-schema's opstellen en bijhorende database normalisatie erop toepassen. In het verleden heb ik ook al in meerdere talen een SQL en/of MySQL server moeten aanspreken. Deze talen zijn Java, .NET, Python en PHP. Mijn optimalisatie kennis beperkt zich tot indexeringen en het vermijden van Select * queries.

Ik heb ook al verschillende keren een LAMP stack opgezet en voor development een SQL-server geïnstalleerd.

PCRE(Perl Compatible Regular Expressions)

Hier heb ik een redelijke basis van. Dingen zoals [A-Z]{3}[0-9]{1,3}["-"]?[^\/]*? zijn voor mij begrijpbaar. Ook capture groups en non-capture groups zijn mij bekend. Enkel dingen zoals 'zero-width negative lookahead' zie niet zo vlot gaan.

GNU/Linux

Voornamelijk kennis van de Debian/Ubuntu familie. Ooit ook eens Gentoo/Funtoo(source based-distro) geïnstalleerd. Hier heb ik een redelijke basis kennis van. Het meest geavanceerde dat ik hier ooit mee gedaan heb is een traffic shaper VM opgezet om latency binnen een ESXi server te verhogen tussen 2 virtuele machines. Ik hoop hier in de toekomst meer details van te kunnen posten.

Programmeer talen

Javascript

Op dit moment van schrijven zowat de taal die ik het best onder de knie heb. Ik heb een redelijke kennis van de DOM en ben in de toekomst van plan te kijken naar NodeJS API en standaard modules. Voor mensen die ook zin hebben om meer van Javascript te leren kan ik Douglas Crockford zijn lezingen aanraden die op Youtube staan. Hij geeft werkelijk excellente voorbeelden van advanced JS zoals verschillende design patterns die je niet voor mogelijk hield in een simpele taal als Javascript.

Java

Zoals iedere MCT student die door PROG IV is heb ik een project gemaakt dat een client en server omvatten die over het netwerk communiceren en aan de server zijde een database moet aanspreken. Hiervoor was kennis van het swing GUI framework nodig, multi-threading, design patterns zoals een singleton, databases aanspreken, netwerk communicatie over TCP/IP. Ik heb toen als kleine bonus ervoor gezorgd dat de client de server op hetzelfde subnet automatisch kon detecteren met een broadcast. Hier moet ik mogelijks ook maar eens een blogpost over maken.

Python

Scripttaal die ik mijzelf aangeleerd heb vanwege zijn simpliciteit. Mijn eerste poging was met Perl maar die is iets minder beginners-vriendelijk. Mijn niveau kun je het best pijlen uit de kleine scriptjes die ik geschreven heb en op mijn Github gepost heb.

Perl

Reguliere expressies is het enige wat nog van mijn Perl kennis over is. (Zie PCRE)

PHP & Wordpress

Ik ben niet echt fan van PHP. Ik kan de volgende blogpost aanraden voor de mensen die het hier met mij oneens zouden zijn. PHP a fractal of bad design

Ondanks dat is er best goede software geschreven in PHP. Mijn ervaring met Wordpress was niet onaangenaam. Al kan ik 1 tip geven. Codeer nooit één enkel PHP bestand met een Byte Order Mark(BOM). Als PHP dergelijke bestanden importeert zal hij (meestal onzichtbare) unicode karakters doorsturen met de HTML. Dit is iets dat extreem moeilijk te vinden is en normaal gezien enkel in oudere versies van PHP zou mogen zitten maar ik heb het in PHP 5.3 nog meegemaakt. De tijd om dit te vinden en te debuggen is mij nooit teruggegeven.

Te Leren

Dingen die ik in de toekomst wil leren en mijn motivatie ervoor.

NGINX

Tegenwoordig een van de beste opensource webservers en reverse-proxy servers. Dit kunnen opzetten en configureren lijkt me de moeite waard om te leren.

FreeBSD

FreeBSD is met ondersteuning voor ZFS te bieden volgens mij een van de beste data-opslag besturingssystemen die er bestaan.

MongoDB

De belangrijkste NoSQL database. Waarschijnlijk combineer het ik het leren van MongoDB met verder leren van de NodeJS standaard modules.
